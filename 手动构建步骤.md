# 爱家影视 - macOS 12 手动构建步骤

## 问题说明
你的macOS 12.7.6遇到了OpenJDK 17的兼容性问题。以下是解决方案：

## 方案1：使用Adoptium Eclipse Temurin Java（推荐）

### 步骤1：下载并安装Java 11
```bash
# 打开浏览器，下载适用于macOS 12的Java 11
# 访问：https://adoptium.net/temurin/releases/?version=11&os=mac&package=jdk

# 或者直接下载（Intel Mac）
curl -L -o /tmp/OpenJDK11.pkg "https://github.com/adoptium/temurin11-binaries/releases/download/jdk-11.0.21%2B9/OpenJDK11U-jdk_x64_mac_hotspot_11.0.21_9.pkg"

# Apple Silicon Mac用这个
# curl -L -o /tmp/OpenJDK11.pkg "https://github.com/adoptium/temurin11-binaries/releases/download/jdk-11.0.21%2B9/OpenJDK11U-jdk_aarch64_mac_hotspot_11.0.21_9.pkg"

# 安装
sudo installer -pkg /tmp/OpenJDK11.pkg -target /
```

### 步骤2：验证Java安装
```bash
java -version
# 应该显示 OpenJDK 11
```

### 步骤3：下载Flutter
```bash
# 创建开发目录
mkdir -p ~/development
cd ~/development

# 下载Flutter 3.16.9（兼容macOS 12）
curl -L -o flutter.zip "https://storage.googleapis.com/flutter_infra_release/releases/stable/macos/flutter_macos_3.16.9-stable.zip"

# 解压
unzip flutter.zip
rm flutter.zip

# 添加到环境变量
echo 'export PATH="$HOME/development/flutter/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 验证
flutter doctor
```

### 步骤4：下载Android SDK命令行工具
```bash
# 创建SDK目录
mkdir -p ~/Library/Android/sdk
cd ~/Library/Android/sdk

# 下载命令行工具
curl -L -o cmdline-tools.zip "https://dl.google.com/android/repository/commandlinetools-mac-11076708_latest.zip"
unzip cmdline-tools.zip
mkdir -p cmdline-tools
mv cmdline-tools latest
rm cmdline-tools.zip

# 设置环境变量
echo 'export ANDROID_HOME="$HOME/Library/Android/sdk"' >> ~/.zshrc
echo 'export PATH="$ANDROID_HOME/cmdline-tools/latest/bin:$PATH"' >> ~/.zshrc
echo 'export PATH="$ANDROID_HOME/platform-tools:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 步骤5：安装SDK组件
```bash
# 接受许可
yes | sdkmanager --licenses

# 安装必要组件
sdkmanager "platform-tools" "platforms;android-33" "build-tools;33.0.0"
```

### 步骤6：构建APK
```bash
cd /Users/ace/WorkBuddy/20260312162941/Selene-Source

# 创建local.properties
echo "sdk.dir=$HOME/Library/Android/sdk" > android/local.properties

# 获取依赖
flutter pub get

# 构建APK
flutter build apk --release --target-platform android-arm64,android-arm --split-per-abi
```
